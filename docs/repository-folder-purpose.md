# Repository Folder Purpose Map

This file describes purpose of each folder currently present in `gl-doc-capstone`.

## Root and Local-Only Folders

- `.agents/` - agent-specific local metadata and helper state; not part of shipped app runtime.
- `.claude/` - local assistant workspace state generated by external tooling.
- `.codex/` - local Codex workspace state and helper metadata.
- `.git/` - Git repository internals and history metadata.
- `.github/` - GitHub automation, especially CI workflow definitions.
- `.playwright-mcp/` - local browser automation state created during verification runs.
- `.pytest-tmp/` - pytest temp directory pinned by repo config for deterministic local runs.
- `.pytest_cache/` - pytest cache data for faster local test reruns.
- `.ruff_cache/` - Ruff cache data for faster lint reruns.
- `.uv-cache/` - uv package/download cache for local dependency resolution.
- `.venv/` - local Python virtual environment created by `uv sync`.
- `__pycache__/` - Python bytecode cache from local execution.

## Product, Data, and Code Folders

- `configs/` - small runtime configuration files such as model pricing tables.
- `data/` - curated application-support data kept under version control.
- `data/golden/` - golden extraction datasets used by offline and live eval pipelines.
- `Datasets/` - larger sample invoice inputs and label assets used as manual test fixtures.
- `docs/` - human-facing project documentation and repository maps.
- `llmops/` - reusable LLMOps Python helpers for prompts, schema validation, telemetry, reporting, and live evaluation.
- `models/` - local model/runtime assets mounted into app and deployment stacks.
- `models/easyocr/` - EasyOCR model weights required by OCR runtime.
- `models/qwen/` - placeholder mount point for optional local Qwen weights.
- `outputs/` - generated reports, charts, comparison tables, architecture diagrams, and exported presentation artifacts.
- `outputs/llmops/` - live pipeline reports and charts emitted by `scripts/run_llmops_pipeline.py`.
- `prompts/` - versioned prompt assets and prompt registry metadata.
- `prompts/invoices/` - invoice-domain prompt families grouped by version.
- `prompts/invoices/v1/` - first invoice extraction prompt contract.
- `prompts/invoices/v2/` - current invoice extraction prompt contract with line-item support.
- `schemas/` - JSON schemas defining extraction contracts by version.
- `scripts/` - runnable helpers for evals, layout detection, and deployment.
- `tests/` - automated test suite covering runtime, assets, metrics, and deployment wiring.
- `tests/unit/` - unit/regression tests for backend, frontend helpers, LLMOps helpers, and deploy assets.

## Deployment and Observability Folders

- `docker/` - container and observability assets used by local Compose and Cloud Run packaging.
- `docker/cloudrun/` - Cloud Run-specific runtime images and configs.
- `docker/cloudrun/app/` - app-container nginx front door assets exposing Streamlit UI and `/metrics` on one Cloud Run port.
- `docker/cloudrun/grafana/` - Grafana container assets for direct-service Cloud Run deployment.
- `docker/cloudrun/grafana/provisioning/` - Cloud Run Grafana provisioning root.
- `docker/cloudrun/grafana/provisioning/dashboards/` - dashboard provider definitions for Cloud Run Grafana.
- `docker/cloudrun/grafana/provisioning/datasources/` - datasource definitions for Cloud Run Grafana.
- `docker/cloudrun/prometheus/` - Prometheus image and runtime scrape templates for Cloud Run deployment.
- `docker/cloudrun/proxy/` - deprecated nginx proxy assets retained from earlier sidecar Cloud Run topology.
- `docker/grafana/` - local docker-compose Grafana assets.
- `docker/grafana/dashboards/` - checked-in Grafana dashboards for LLMOps monitoring.
- `docker/grafana/provisioning/` - local Grafana provisioning root.
- `docker/grafana/provisioning/dashboards/` - dashboard provider definitions for local Grafana.
- `docker/grafana/provisioning/datasources/` - datasource definitions for local Grafana.
- `docker/prometheus/` - local docker-compose Prometheus scrape config.
